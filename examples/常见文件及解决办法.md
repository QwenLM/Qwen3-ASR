# 分片教程

分片后能显著降低加载所需内存（官方仓库加载至少需要4G内存，对于8G以下电脑，会动用虚拟磁盘，很伤硬盘，分片后内存使用峰值大概为 max_shard_size，按下面分片也就 1G）

1. 准备好官方仓库或自己训练后的仓库

2. 编辑原仓库 `generation_config.json` 文件：  
   去掉 `"temperature": 0.000001`，它和 `"do_sample": false` 冲突，asr 不需要这个参数，不去掉会分片失败

3. 分片脚本

```python
import os

# 设置环境变量（等效于 Windows 的 set 命令）
os.environ["HF_HOME"] = r"E:\Workspace\Python\Asr\cache"
os.environ["XDG_CACHE_HOME"] = r"E:\Workspace\Python\Asr\cache"
os.environ["TORCH_HOME"] = r"E:\Workspace\Python\Asr\cache\torch"
os.environ["MODELSCOPE_CACHE"] = r"E:\Workspace\Python\Asr\cache\modelscope"
os.environ["MODELSCOPE_HOME"] = r"E:\Workspace\Python\Asr\cache\modelscope"
# os.environ["TRANSFORMERS_VERBOSITY"] ="info"

# 原始方式加载模型
import torch
from qwen_asr import Qwen3ASRModel
model = Qwen3ASRModel.from_pretrained(
    r"E:\Workspace\Python\Asr\cache\Qwen3-ASR-1.7B",
    dtype=torch.bfloat16,
    device_map="cuda:0",
    max_inference_batch_size=-1,            # 不限制批处理大小
    max_new_tokens=10000000000              # 原始值很小
)

print("开始分片")
model.model.save_pretrained(r"E:\Workspace\Python\Asr\cache\out", max_shard_size="1GB")
print("分片保存成功")

4. 手动合并仓库
    删除官方仓库 model.safetensors.index.json、safetensors 文件
    只复制分片后 out 下的 model.safetensors.index.json、safetensors 文件到官方仓库，其他不要

5.按 Qwen3 asr 官方加载方式继续使用即可
